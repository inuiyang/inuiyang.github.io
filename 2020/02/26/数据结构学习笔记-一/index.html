数据结构介绍

抽象数据类型(ADT）

  • 抽象数据类型 (ADT): 是一种描述数据结构的程序设计语言无关的模型。
  • ADT定义为一组数值，和一组相关的操作。

例子：

c++

仅仅描述了输入和输出。
而没有实现的细节

数据结构的实现

  • 决定如何去存储和组织数据.

  • 决定如何去实现相关的操作.

    • 每个ADT描述的操作可以实现为一个或多个函数的形式。

增长率分析

  • T(n) : 运行时间关于输入数据规模大小n的函数。

  • S(n): 消耗空间关于输入数据规模大小n的函数。

渐进分析

T(n) 通常是带有多个参数的复杂表达式

如 Ex: T(n) = c1n2+ c2logn + ….

渐进分析是对T(n)的一种简化分析，该分析保留了T(n)的渐进增长趋势

大O分析 (上界分析)

定义: 对于 非负函数T(n), 如果存在正常数c 和 n0使得当n > n0时 T(n) ≤ cf(n)，则称 T(n) 属于集合O(f(n))。或记为(T(n) = O(f(n))。

即T(n)的值永远不超过f(n)的常数倍。

大 Omega 分析(下界分析)

定义:对于 非负函数T(n), 如果存在正常数c 和 n0使得当n > n0时 T(n) ≥ cg(n)，则称 T(n) 属于集合 W(g(n)) 。或记为(T(n) = W(g(n)) 。

大 Theta 分析

定义: 若T(n) =O(h(n)) 且T(n) =W(h(n))，则称T(n) =Q(h(n)) 。

简化规则

常数规则

If f(n) = O(kg(n)) for any constant k > 0, then f(n) = O(g(n)).

eg. 3n = O(3n) = O(n) .

对下界、Theta也适用。

加法规则

If f1(n) = O(g1(n)) and f2(n) = O(g2(n)), then f1 (n) + f2(n) = O(max(g1(n), g2(n))).

eg. n^2 + n = O(n^2).

eg. 3n^2 + 2n + 4 = O(3n^2) = O(n^2).

对下界、Theta也适用。

其他规则

  1. If f(n) = O(g(n)) and g(n) = O(h(n)), then f(n) = O(h(n)).

  2. If f1(n) = O(g1(n)) and f2(n) = O(g2(n)) then f1(n)f2(n) = O(g1(n)g2(n)).

对下界、Theta也适用。

线性表

线性表的定义

  • 线性表是由有限的数据元素构成的序列。

  • 每个数据元素具有特定的数据类型。

    • 线性表支持的操作包括：
    • 添加数据元素
    • 删除数据元素
    • 查找数据元素求表中元素个数

  • 当前位置：在线性表中插入、删除、或读去数据的位置。

    • 当前位置通过栅栏 (fence)来定义。栅栏将线性表划分为左右两个部分。插入、删除、读取都在栅栏所在的位置执行。

    • 例如，带有栅栏的线性表可表示如下:

      <12, 22 | 32, 15,29>

  • 一个包含0个数据元素的线性表称为空表。

  • 线性表的长度指的是线性表中数据元素的个数。

  • 线性表的开始端称为表头（head）, 线性表的结束端称为表尾（tail）.

  • 有序表：表中的所有的数据元素是根据其数值的大小来进行先后排列。

  • 无序表：表中数据元素的位置和该元素的数值没有必然联系。

线性表的基本实现方法

数组

  • 插入和删除：Q(n).
  • 求前驱和随机访问： Q(1).
  • 需要提前为数组分配空间
  • 当数组是满的时候，没有额外的开销

连续存储，进行插入的时候要将 fence 后的全部元素往后挪，删除时要往前挪。

链表

  • 插入和删除：Q(1).
  • 求前驱和随机访问：Q(n).
  • 消耗的空间随数据元素个数的增长而增长
  • 每个数据元素的存储需要额外的空间开销

数据不需要连续存放，每个元素结点用一个指针指向其后续元素（next）。

栈（stack）

栈的定义

定义：插入、删除只在一端进行的线性表。

LIFO 性质: Last In, First Out. （后进先出）

  • 插入元素到栈中: 入栈(PUSH)
  • 从栈中取出元素: 出栈(POP)
  • 栈中插入、删除的位置：栈顶(TOP).

队列（queue）

队列的定义

定义：插入和删除分别在表的两端进行的线性表.

FIFO 性质: First in, First Out （先进后出）

  • 插入: enqueue（入队）

  • 删除: dequeue （出队）

  • 第一个元素: front（队头）

  • 最后一个元素: rear（队尾）
